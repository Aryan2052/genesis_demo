💡 Why We Store Data -- Use Cases

🔍

Historical Analysis

Query past events to discover patterns, trends, and recurring behaviors. E.g., "Show all large movements in the last 24 hours."

📋

Audit Trail

Immutable off-chain record of all monitored activity. Local DB has every event indexed and instantly searchable.

📊

Dashboard Data Source

This dashboard is powered by SQLite. Charts, tables, and stats are populated from stored data -- not by re-scanning the blockchain.

🚫

Alert Deduplication

Before sending a Telegram alert, check if a similar alert was sent recently. Prevents spam by querying the alerts table.

📉

Anomaly Detection

Statistical analysis (z-scores, moving averages) requires historical data points. Store events, compute baselines, detect outliers.

💰

Cost Savings

RPC calls are expensive. Once stored locally, future queries hit SQLite (free, instant) instead of making new RPC calls.
